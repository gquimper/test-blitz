Form & fields visual treatment developed for all Bell platform and website forms. You will find various examples of fields, option buttons and tools that you can use to design the forms page.

  • Field outline rounded corners = 4px
  • Field outline color = #D4D4D4
  • Placeholder text color = #555555
  • Input text color = #111111

Tooltips

A tooltip is a type of popup that provides additional information to users once they hover over with their mouse for a configurable delay period. When the mouse leaves the target area, the tooltip goes away.

This type of element will often be used on desktop views and not generally recommended on mobile views, where hover behaviour can be inconsistent.

The element will be placed after a line of text or a field.

Form structure - Desktop

The questions are right-aligned and the fields and options should be left-aligned to ensure readability.

  • Minimum margin of 30 px
  • Inside padding of 30 px between elements
  • 1px outline separator, color #D4D4D4

Form structure - Mobile

Every components is left aligned. The radio buttons and checkboxes are in boxes to help the user click on the desired option(s).
Unless it is for a really short content required (i.e.: ZIP code), the field box should be the width of the screen size.

Stacked buttons opt on mobile

Error treatments can be presented on forms, flows where users may enter incomplete, incorrect or have missed information.

  • Error red color #BD2025
  • Inside padding of 15 px between elements
  • Minimum margin of 15 px
  • 1px outline box, color #D4D4D4
